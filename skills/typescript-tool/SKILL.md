# TypeScript Tool — Substrate Builder

## Overview

TypeScript CLI tools built hermetically using pleme-linker (Nix-native npm
dependency resolution). No npm/pnpm runs in the Nix sandbox. Dependencies are
fetched individually via `deps.nix`, then linked by pleme-linker.

Supports workspace packages (pre-built or from-source), TypeScript compilation,
and CLI wrapper script generation.

**Builder:** `substrate/lib/typescript-tool.nix` (via `mkTypescriptTool`)
**Dep resolver:** pleme-linker (Rust-based npm resolver for Nix)

## flake.nix Template

```nix
{
  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-25.11";
    substrate = {
      url = "github:pleme-io/substrate";
      inputs.nixpkgs.follows = "nixpkgs";
    };
    pleme-linker = {
      url = "github:pleme-io/pleme-linker";
      inputs.nixpkgs.follows = "nixpkgs";
    };
  };

  outputs = { self, nixpkgs, substrate, pleme-linker, ... }:
    let
      systems = [ "aarch64-darwin" "x86_64-linux" "aarch64-linux" ];
      forEachSystem = f: nixpkgs.lib.genAttrs systems (system:
        let
          pkgs = import nixpkgs { inherit system; };
          tsBuilders = import "${substrate}/lib/typescript-tool.nix" { inherit pkgs; };
          plemeLinkerBin = tsBuilders.mkPlemeLinker {
            plemeLinkerSrc = pleme-linker;
          };
        in f pkgs tsBuilders plemeLinkerBin
      );
    in {
      packages = forEachSystem (pkgs: tsBuilders: plemeLinkerBin: {
        default = tsBuilders.mkTypescriptTool {
          name = "{tool-name}";
          src = ./src;
          cliEntry = "cli.js";
          binName = "{tool-name}";
          plemeLinker = plemeLinkerBin;
          # Optional:
          # parentTsconfig = ./tsconfig.json;
          # workspaceDeps = { "@myorg/shared" = sharedBuild; };
        };
      });

      devShells = forEachSystem (pkgs: _: _: {
        default = pkgs.mkShell {
          buildInputs = [ pkgs.nodejs_20 pkgs.nodePackages.typescript ];
        };
      });
    };
}
```

## Builder Variants

### `mkTypescriptTool`
Build a CLI tool from a single package with pre-built workspace deps.

| Argument | Required | Default | Description |
|----------|----------|---------|-------------|
| `name` | yes | — | Package name |
| `src` | yes | — | Source directory containing `deps.nix` |
| `cliEntry` | yes | — | JS entry point (e.g., `"cli.js"`) |
| `binName` | yes | — | Binary name for the wrapper script |
| `plemeLinker` | yes | — | pleme-linker derivation |
| `parentTsconfig` | no | `null` | Path to parent tsconfig.json |
| `workspaceDeps` | no | `{}` | Pre-built workspace packages `{ name = derivation; }` |

### `mkTypescriptToolWithWorkspace`
Build a CLI tool with workspace packages built from source.

| Argument | Required | Default | Description |
|----------|----------|---------|-------------|
| `name` | yes | — | Package name |
| `src` | yes | — | Source directory |
| `cliEntry` | yes | — | JS entry point |
| `binName` | yes | — | Binary name |
| `plemeLinker` | yes | — | pleme-linker derivation |
| `workspaceSrcs` | no | auto | List of `{ name, src }` for workspace packages |
| `workspaceRoot` | no | `null` | Root for resolving workspace package paths |

### `mkTypescriptToolAuto`
Auto-discovers everything from `package.json` — most minimal interface.

| Argument | Required | Default | Description |
|----------|----------|---------|-------------|
| `src` | yes | — | Source directory with `package.json` |
| `plemeLinker` | yes | — | pleme-linker derivation |
| `workspaceRoot` | no | `null` | Root for workspace packages |

### `mkTypescriptPackage`
Build a TypeScript library package (no CLI wrapper).

| Argument | Required | Default | Description |
|----------|----------|---------|-------------|
| `name` | yes | — | Package name |
| `src` | yes | — | Source directory |
| `plemeLinker` | yes | — | pleme-linker derivation |
| `workspaceDeps` | no | `{}` | Pre-built workspace packages |

## deps.nix Format

Each TypeScript package needs a `deps.nix` generated by pleme-linker:

```nix
{
  packages = {
    "typescript-5.7.3" = {
      pname = "typescript";
      version = "5.7.3";
      url = "https://registry.npmjs.org/typescript/-/typescript-5.7.3.tgz";
      integrity = "sha256-...";
      dependencies = [];
      hasBin = true;
    };
    # ... more packages
  };
  workspacePackages = [];
}
```

## Initial File Layout

```
{tool-name}/
  flake.nix
  tsconfig.json
  package.json
  src/
    cli.ts
    deps.nix         # Generated by pleme-linker
```

## Setup Steps

1. Create repo and clone:
   ```bash
   gh repo create pleme-io/{tool-name} --public
   cd ~/code/github/pleme-io && git clone git@github.com:pleme-io/{tool-name}.git
   cd {tool-name}
   ```

2. Initialize the project:
   ```bash
   npm init -y
   npx tsc --init
   ```

3. Generate `deps.nix`:
   ```bash
   npm install
   nix run github:pleme-io/pleme-linker -- generate-deps ./
   ```

4. Create `flake.nix` using the template above.

5. Verify:
   ```bash
   nix build .#default
   nix run .#default -- --help
   ```

## Regeneration

Use `mkTypescriptRegenApp` to create a regen app:

```nix
apps.regen = {
  type = "app";
  program = toString (tsBuilders.mkTypescriptRegenApp {
    name = "{tool-name}";
    plemeLinker = plemeLinkerBin;
    projectDirs = [ ./src ];
  });
};
```

## Conventions

- `deps.nix` must be committed to git — pleme-linker generates it once
- All npm dependencies are fetched as individual tarballs by Nix (no network in sandbox)
- Node.js 20 is the default runtime
- Workspace deps can be pre-built (separate derivations) or built from source
- CLI wrapper uses `pkgs.nodejs_20` for the shebang

## Anti-Patterns

- Never run `npm install` inside a Nix build — use pleme-linker
- Never use `buildNpmPackage` with pleme-linker projects — they use different dep strategies
- Never skip `deps.nix` regeneration after adding new npm dependencies
